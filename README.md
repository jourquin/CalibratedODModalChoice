# Calibrated OD Modal-Choice Framework for European Multimodal Freight Models

This repository contains the input data, Python/Biogeme estimation script, main results, and a [Nodus](https://github.com/jourquin/Nodus) modal-choice plugin associated with the calibrated modal-choice framework implemented in the Python script.

The framework estimates a freight modal-choice model for road, inland waterway transport (IWW), and rail using aggregate OD-commodity matrices. It combines:

1. a behavioural multinomial logit model estimated with Biogeme;
2. a bounded base-year calibration layer that improves the reproduction of modal OD matrices;
3. OD-cell validation based on the WAPE metric and bias;
4. an implementation in the Nodus assignment environment through a specific plugin.

The purpose of the framework is not only to reproduce aggregate modal shares, but also to generate modal OD matrices that are suitable for network assignment.

## Model overview

The model is estimated on aggregate OD-commodity observations. For each OD-commodity cell, the input data contain observed tonnes by mode, together with modal attributes such as cost, distance, moving duration, and mode availability.

Road is used as the reference mode. IWW and rail utilities are expressed relative to road, using logarithmic ratios for cost and moving time. Distance is introduced through mode-specific piecewise-linear splines. Commodity-group-specific constants capture differences in modal propensity by commodity.

The behavioural model alone is useful for capturing systematic modal-choice effects, but it does not fully reproduce the sparse geography of rail and IWW matrices. The calibration layer therefore adds bounded utility corrections to the non-road alternatives. These constants are interpreted as base-year correction terms, not as behavioural coefficients.


## Input data

The `Input data/` directory contains MySQL/MariaDB dumps for the three input tables used by the Python script:

```text
biogeme_input_model1.sql
biogeme_input_model2.sql
biogeme_input_model3.sql
```

These correspond to the three empirical datasets:

```text
model 1: Europe_NUTS2
model 2: Benelux+_NUTS3
model 3: Germany_NUTS3
```

To import the data into a MySQL or MariaDB database:

```bash
mysql -u <user> -p <database_name> < "Input data/biogeme_input_model1.sql"
mysql -u <user> -p <database_name> < "Input data/biogeme_input_model2.sql"
mysql -u <user> -p <database_name> < "Input data/biogeme_input_model3.sql"
```

The Python script expects the tables to be named:

```text
biogeme_input_model1
biogeme_input_model2
biogeme_input_model3
```


## Running the Python/Biogeme script

The estimation script is located in:

```text
Python Biogeme script/CalibratedODModalChoice.py
```

The helper functions are stored in:

```text
Python Biogeme script/BiogemeUtils.py
```

The script requires Python and the main scientific Python stack, together with Biogeme and a MySQL/MariaDB connector. A typical environment includes:

```text
biogeme
numpy
pandas
pymysql
```

The exact parameters used to generate the results included in this repository are documented in:

```text
Results/run_parameters.md
```

A typical command has the following form:

```bash
python "Python Biogeme script/CalibratedODModalChoice.py" \
  --models 1,2,3 \
  --user <mysql_user> \
  --password <mysql_password> \
  --host 127.0.0.1 \
  --database <database_name> \
  --output-dir output \
  --calibration-level mode_od_group \
  --store-parameters-in-table \
  --parameter-table-prefix modal_choice_params
```

Users should adapt the database credentials, output directory, and runtime options to their local installation.

The script produces estimation results, validation reports, calibration constants, comparison tables, and optional database parameter tables for the Nodus plugin.

## Nodus plugin

The `Nodus plugin/` directory contains:

```text
CalibratedODModalChoice.java
CalibratedODModalChoice.jar
```

The Java plugin allows the calibrated modal-choice model estimated in Python to be applied inside the Nodus network-assignment environment.

The plugin reads the estimated parameters and calibration constants from a database table generated by the Python script. This avoids manually inserting large numbers of coefficients and calibration constants into a Nodus `.costs` file.

### Installing the plugin

Copy the compiled ``CalibratedODModalChoice.jar`` into the relevant Nodus project directory used by your Nodus installation.

The Java source file is also provided so that the plugin can be inspected, modified, or recompiled if needed.

### Activating the parameter table

The Nodus `.costs` file must contain the name of the parameter table generated by the Python script. For example:

```text
@paramTable=modal_choice_params_model1
```

or, depending on the table name used when running the Python script:

```text
@paramTable=<your_parameter_table_name>
```

The plugin reads this table before applying the modal-choice model. The parameter table must be available in the database used by the Nodus project.


## Reproducing the workflow

A complete reproduction workflow is:

1. Create a MySQL or MariaDB database.
2. Import the three SQL dumps from `Input data/`.
3. Install the required Python packages.
4. Run `CalibratedODModalChoice.py` using the parameters documented in `Results/run_parameters.md`.
5. Check the generated output files.
6. If using Nodus, copy the JAR plugin into the Nodus project/plugin directory.
7. Add the `@paramTable` entry to the `.costs` file.
8. Run the Nodus assignment with the calibrated modal-choice plugin.
